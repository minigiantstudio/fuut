  <files>apps/web/vite.config.ts, apps/web/src/App.tsx</files>
  <action>
    Driven by the Task 1 treemap; only apply what the data points at. Likely candidates:

    1. **Route code-splitting** for the public `/s/:token` page and the
       `/admin` subtree:
       ```ts
       const SnapshotPage = lazy(() => import('./routes/Snapshot.tsx'));
       const AdminDashboard = lazy(() => import('./admin/AdminDashboard.tsx'));
       const AdminLogin = lazy(() => import('./admin/AdminLogin.tsx'));
       ```
       Wrap each lazy route in `<Suspense fallback={<MinimalLoader />}>`.
       Expected saving: ~80KB gzipped off the home payload
       (`html-to-image` alone is ~40KB).
    2. **Icon-lib tree-shake** — `lucide-react` is huge if imported as
       barrel. Confirm Vite tree-shakes via the existing per-icon imports
       (`import { Pencil, Trash2 } from 'lucide-react'`). If not, switch to
       direct file imports: `import Pencil from 'lucide-react/dist/esm/icons/pencil'`.
    3. **Defer html-to-image dynamic import** within RankingTab's
       `handleShare`:
       ```ts
       const { toPng } = await import('html-to-image');
       ```
       Pulls 40KB out of the Ranking-tab initial render and only loads it on
       first Share click.
    4. **Vite manualChunks** — group infrequently-changed vendor code
       (`react`, `react-dom`, `@tanstack/react-query`, `@supabase/supabase-js`)
       into a stable `vendor` chunk so users get cache hits on subsequent
       deploys.

    Re-measure after each change. Stop when:
    - Lighthouse Performance ≥ 85 on `Mobile - Slow 4G`.
    - Total home-route gzipped JS < 350KB.
    Otherwise iterate on the next biggest module.
