       - RLS: Owner read/write; No other public access.

<task type="auto">
  <name>Task 3: Supabase Auth Middleware</name>
  <files>apps/api/src/middleware/auth.ts, apps/api/src/index.ts</files>
  <action>
    Implement `authMiddleware` using the Supabase **anon** key:
    - Extract Bearer token from `Authorization` header.
    - Call `supabase.auth.getUser(token)` to verify JWT. This ensures the token was issued by our Supabase project and is currently valid.
    - Attach `user` to `req.user`.
    - Apply middleware to protected route `/api/me`.
    
    Rationale: Using `service_role` key would bypass RLS; `anon` key is appropriate for verifying user-provided JWTs.
  </action>
  <verify>
    <automated># Test with an invalid token
    curl -I -H "Authorization: Bearer invalid" http://localhost:3001/api/me | grep 401</automated>
  </verify>
  <done>Backend securely validates Supabase JWTs via middleware</done>
