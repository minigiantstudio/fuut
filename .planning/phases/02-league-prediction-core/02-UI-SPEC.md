# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2: League & Prediction Core. Generated by
> gsd-ui-researcher, verified by gsd-ui-checker. All decisions are pre-populated from
> CONTEXT.md (D-01–D-11), RESEARCH.md, and direct codebase analysis of existing components.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (default style, slate base, css-variables) |
| Preset | Detected from `apps/web/components.json` — no custom preset string; standard init |
| Component library | Radix UI (via shadcn/ui wrappers) |
| Icon library | lucide-react (already imported in PredictTab — `X` icon) |
| Font | Press Start 2P (Google Fonts, monospace pixel art) — applied globally via `body` rule |

**Aesthetic mandate (DEC-003):** Retro 8-bit pixel art. Zero border-radius (`--radius: 0px`).
`image-rendering: pixelated` on all elements. Pixel box-shadows via `.pixel-border`,
`.pixel-border-sm`, `.pixel-inset`. Blinking cursor on inputs via `@keyframes pixel-blink`.
Press effect via `.pixel-press` (translate on `:active`, collapse shadow).
Screen transitions via `.pixel-screen-enter` (step-end slide-in) and `.pixel-fade-enter`.

**New surfaces in scope:** Onboarding fork (step 1 split) → create-league step machine
(4 new steps) → create-confirm screen → LeagueSwitcher component → TopBar switcher trigger
→ LeagueTab regenerate-code button + dialog → PredictTab "Locks in" countdown label.

## Spacing Scale

Tailwind spacing is used directly (1 unit = 4px). All values conform to the 4-point grid.

| Token | Value | Usage in Phase 2 |
|-------|-------|------------------|
| xs | 4px (`gap-1`, `p-1`) | Icon-to-label gaps, badge inner padding |
| sm | 8px (`gap-2`, `p-2`) | Between CTA buttons, input field margin |
| md | 16px (`px-4`, `p-4`) | Card internal padding, input horizontal padding |
| lg | 24px (`space-y-6`, `py-6`) | Step screen vertical rhythm between sections |
| xl | 32px (`py-8`, `space-y-8`) | Onboarding screen top/bottom padding (matches existing `space-y-8` in steps) |
| 2xl | 48px (`py-12`) | Full-screen onboarding outer padding (matches existing `py-12`) |
| 3xl | 64px | Not used in Phase 2 surfaces |

Exceptions:
- TopBar height: `var(--topbar-height)` = 3.5rem (56px) — CSS custom property, not overridden.
- Nav height: `var(--nav-height)` = 4.5rem (72px) — CSS custom property, not overridden.
- Match card score inputs: 32px × 32px (`w-8 h-8`) — fixed square, pixel-inset border — pre-existing pattern, preserved.
- Touch target minimum: all tappable elements must be at least 44px tall (`h-11` or `h-12`). The existing button convention uses `h-12` (48px); apply this to all new primary/secondary buttons in Phase 2.
- Invite code display in confirm screen: 48px height (`py-3 px-4` on the `pixel-inset` container) — matches existing `LeagueTab` invite code card.

## Typography

All text uses `Press Start 2P` (monospace). The font has no italic or bold variants —
weight declarations are effectively ignored by the renderer; `font-bold` has no visual
effect. The size hierarchy below is expressed in pixel sizes matching existing conventions.

**Maximum 4 named sizes on the global scale. All other surface-specific exceptions are
annotated inline in the surface contracts and do not appear in this table.**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28–30px (`text-3xl`) | 400 (sole weight) | 1.2 | Hero heading on step 1 — "Fuut 2026" title |
| Heading | 12px (`text-[12px]`) | 400 | 1.3 | Step screen titles ("Name your league", "Your nickname") |
| Body | 8px (`text-[8px]`) | 400 | 1.5 | Button labels, form labels, member list nicknames |
| Caption | 6px (`text-[6px]`) | 400 | 1.4 | Error messages, status captions, countdown label, badge text, sub-labels, helper text ("How others see you in…") |

Notes:
- The `text-base` class (16px) appears on the nickname input value text (`text-base`) — this is the user's typed content, not a UI label. Preserve this convention for input value text.
- The `text-xs` class (12px) is used for member names and match team names — preserve as the body-level readable text in list rows.
- **Surface-specific exception — invite code display only:** `text-[14px] tracking-[0.3em]` on the 4-char code in the `pixel-inset` card (Surface 2, create-confirm step, and LeagueTab). This is a one-off display override; it is NOT a named scale entry and must not be used elsewhere.
- **Surface-specific exception — step sub-labels:** `text-[7px]` appears in existing Onboarding.tsx sub-label patterns. Where this pattern is replicated (create-name, create-nickname sub-labels), preserve `text-[7px]`; it is a surface-level detail, not a global scale entry.
- Never introduce `text-sm` (14px) as a label; reserve 14px solely for the invite code display exception above.

## Color

All colors reference CSS custom properties defined in `apps/web/src/index.css`. Use Tailwind
semantic classes, not raw hex values.

| Role | CSS Variable | Tailwind Class | Value (HSL) | Usage |
|------|-------------|----------------|-------------|-------|
| Dominant (60%) | `--background` | `bg-background` | 200 30% 90% (light blue-grey) | Full-screen onboarding bg, main app bg |
| Secondary (30%) | `--card` | `bg-card` | 0 0% 96% (off-white) | All card surfaces: invite code card, match cards, member list, switcher panel |
| Accent (10%) | `--accent` / `--pixel-gold` | `text-accent`, `bg-pixel-gold` | 45 100% 50% (gold/yellow) | Reserved for accent uses listed below |
| Destructive | `--destructive` / `--pixel-red` | `bg-destructive`, `text-pixel-red` | 0 80% 55% (red) | Error text, destructive actions only |

**Accent reserved for (explicit list):**
1. TopBar league name label — `text-accent` (existing `⚽ {leagueName}` span)
2. OPEN match status badge — `bg-pixel-gold text-foreground` (existing convention)
3. NEEDS_RESULT match status badge — `bg-pixel-gold text-foreground` (existing convention)
4. Share invite link icon/label color — `text-lime-700` (existing convention in LeagueTab; preserve)

**Pixel semantic palette (supplementary, for status only):**
| Token | Class | Value | Reserved for |
|-------|-------|-------|-------------|
| pixel-green | `bg-pixel-green`, `text-pixel-green` | 140 60% 40% | SAVED badge, "Join league" button bg, "Start predicting" button bg, step 1 hero title |
| pixel-blue | `bg-pixel-blue`, `text-pixel-blue` | 210 70% 50% | ADMIN badge, "Let's go" button bg (nickname step), "Enter result" link |
| pixel-gold | `bg-pixel-gold`, `text-pixel-gold` | 45 100% 50% | OPEN/RESULT badges (accent role above) |
| pixel-red | `bg-pixel-red`, `text-pixel-red` | 0 80% 55% | Error messages, "Locks in" countdown label |

**TopBar background:** `bg-green-800` (existing hardcoded Tailwind color) — preserve, do not migrate to a CSS variable in this phase.

**No dark mode:** The app has no dark mode. Do not add `dark:` variants to Phase 2 surfaces.

## New UI Surfaces — Interaction Contracts

### Surface 1: Onboarding Step 1 Fork

**What changes:** Step 1 currently shows one CTA ("Join league"). Phase 2 adds a second CTA
("Create a league") above the invite code input. The invite code input and "Join league" button
remain unchanged.

**Layout order (top to bottom):**
1. Trophy icon + "Fuut 2026" heading + subtitle (unchanged)
2. Avatar strip (unchanged)
3. NEW: "Create a league" button — full-width, `h-12`, `pixel-border`, `bg-foreground text-primary-foreground`
4. Divider: `<p class="text-[6px] text-muted-foreground text-center">— or join with a code —</p>`
5. Invite code input (unchanged — `pixel-inset`)
6. "Join league" button (unchanged — `bg-green-800`)
7. "I played before" recovery link (unchanged)

**"Create a league" button styling:** `bg-foreground text-primary-foreground` (dark/primary fill,
contrasts with the green "Join league" button). `text-[8px] uppercase tracking-wider`. Same
height and pixel-border as all other primary buttons.

**Tap target:** Both CTAs are `h-12` (48px) — meets 44px minimum.

### Surface 2: Create-League Step Machine (4 new steps)

All new steps share the existing screen scaffold:
```
min-h-screen bg-background flex justify-center
  w-full max-w-[430px] flex flex-col px-6 py-12
    flex-1 flex flex-col justify-center space-y-8
```

Step machine states (extend existing `useState<1 | 2 | 3 | 4 | "recovery">`):
Add: `"create-name" | "create-nickname" | "create-email" | "create-confirm"`

**Step "create-name" — Name Your League:**
- Heading: `text-[12px] text-foreground` — "Name your league"
- Sub-label: `text-[7px] text-muted-foreground` — "Your friends will see this name" (surface-specific `text-[7px]`, not a global scale entry)
- Input: `pixel-inset bg-card h-12 px-4 text-base text-foreground` placeholder "League name..." (same pattern as nickname input in step 2)
- Primary CTA: "Next →" — `h-12 pixel-border bg-pixel-blue text-primary-foreground text-[8px] uppercase tracking-wider`
- Disabled when input empty
- Back: `text-[6px] text-muted-foreground` link "← Back" returns to step 1

**Step "create-nickname" — Your Nickname:**
- Reuses the existing step 2 (nickname) layout verbatim
- Heading: "Your nickname"
- Sub-label: `text-[7px] text-muted-foreground` — "How others see you in {newLeagueName}" (surface-specific `text-[7px]`, not a global scale entry)
- Primary CTA: "Next →" (blue) instead of "Let's go" to maintain forward flow
- Back: "← Back" returns to "create-name"

**Step "create-email" — Your Email (mandatory for creator):**
- Heading: `text-[12px] text-foreground` — "Your email"
- Sub-label: `text-[7px] text-muted-foreground` — "Required to manage your league" (surface-specific `text-[7px]`, not a global scale entry)
- Input: `pixel-inset bg-card h-12 px-4 text-[8px]` type="email" placeholder "name@example.com"
- Primary CTA: "Create league" — `h-12 pixel-border bg-pixel-green text-primary-foreground text-[8px] uppercase tracking-wider`
- Disabled when email invalid (must contain `@`) OR when loading
- Loading state: button label "Creating..." — no spinner, consistent with `validateCode` loading pattern
- Error state: `text-[6px] text-pixel-red text-center` below button
- Back: "← Back" returns to "create-nickname"
- No "Skip for now" — email is mandatory for the creator (D-02)

**Step "create-confirm" — League Created:**
- Heading: `text-[12px] text-foreground text-center` — "League created!"
- Trophy icon (same `w-16 h-auto` as step 1) — centered
- Invite code card (full replication of `LeagueTab` invite code card):
  - Outer: `pixel-border bg-card p-4 space-y-3`
  - Label: `text-[6px] text-muted-foreground uppercase` — "Your invite code" (Caption scale)
  - Code display: `pixel-inset bg-background py-3 px-4 text-center` — `text-[14px] tracking-[0.3em] text-foreground` — shows the generated code (surface-specific exception — invite code display only; see Typography notes)
  - Share button: `flex items-center gap-2 mx-auto text-xs text-lime-700` — "📤 Share invite link" (identical to `LeagueTab.handleShare` — uses `navigator.share()` with `navigator.clipboard.writeText()` fallback)
- Primary CTA: "Start predicting" — `h-12 pixel-border bg-pixel-green text-primary-foreground text-[8px] uppercase tracking-wider` — calls `refreshSession()` then enters app
- Loading state: "Loading..." (while `refreshSession` resolves — per Pitfall 3 in RESEARCH.md, show confirm screen immediately from RPC response, enter app only after session resolves)
- Screen transition: `.pixel-screen-enter` on the confirm container

### Surface 3: LeagueSwitcher Component

**Trigger:** The league name in TopBar becomes a `<button>`. Add a small downward caret indicator.

**TopBar trigger (replaces static `<span>`):**
```
<button onClick={() => setDrawerOpen(true)} className="flex items-center gap-1 text-[10px] font-bold text-accent pixel-press">
  ⚽ {leagueName} <span className="text-[7px] opacity-70">▾</span>
</button>
```
The `pixel-press` class gives the tactile press effect on tap.

**Drawer contents (LeagueSwitcher component):**
- Title: `text-[8px] text-foreground uppercase tracking-wider` — "Your leagues"
- League list: one row per league in `session.leagues[]`
  - Row height: `h-12` (48px minimum touch target)
  - Row layout: `flex items-center gap-2 px-4 py-3`
  - League name: `text-[8px] text-foreground flex-1`
  - Active indicator: if `leagueId === session.activeLeagueId`, show badge `text-[6px] px-2 py-1 bg-pixel-green text-primary-foreground border-2 border-foreground` — "ACTIVE"
  - Role badge: if `role === "admin"`, show `text-[6px] px-2 py-1 bg-pixel-blue text-primary-foreground border-2 border-foreground` — "ADMIN"
  - Divider: `divide-y-2 divide-foreground` on the list container (matches member list in LeagueTab)
  - Tap row to call `setActiveLeague(leagueId)` and close drawer
- Outer wrapper: `pixel-border bg-card` on the list container
- Drawer padding: `px-4 pb-8 pt-4`

**Empty/single-league state:** If `session.leagues.length <= 1`, do not render the `▾` caret and do not open a drawer on tap. The league name remains static text. (Users with one league have no switching to do.)

### Surface 4: LeagueTab Admin — Regenerate Invite Code

**Location:** Inside the existing `{isAdmin && ...}` admin section in `LeagueTab.tsx`. Added as a new row in the `pixel-border bg-card divide-y-2 divide-foreground` manage block, below the "Rename league" button (which is deferred to Phase 4 but the container already exists).

**Button row:**
```
<button className="flex items-center gap-3 px-3 py-2 w-full text-left">
  <span className="text-foreground text-xs">↺ Regenerate invite code</span>
</button>
```
Uses `text-xs` (12px) matching the existing "Rename league" row text size. `py-2` = 8px vertical padding (4-point grid).

**Confirmation dialog (using `dialog.tsx`):**
- Trigger: tapping "Regenerate invite code" row opens a shadcn/ui `<Dialog>`.
- Dialog heading: `text-[10px] text-foreground uppercase` — "Regenerate code?"
- Dialog body: `text-[6px] text-muted-foreground` — "The old code stops working immediately. Anyone with the current link will need the new code." (Caption scale)
- Two buttons (inside `DialogFooter`):
  - Cancel: `h-10 px-4 pixel-border-sm text-[6px] text-foreground bg-card` — "Keep current code"
  - Confirm (destructive): `h-10 px-4 pixel-border-sm text-[6px] text-primary-foreground bg-destructive` — "Regenerate"
- On confirm: call `supabase.rpc("regenerate_invite_code", { p_league_id: session.leagueId })` → `queryClient.invalidateQueries({ queryKey: ["league", session.leagueId] })`
- Loading state on Confirm button: "Regenerating..." — disable both buttons while pending
- Error state: `text-[6px] text-pixel-red text-center` inside dialog body below description

### Surface 5: PredictTab — "Locks in Xh Ym" Countdown

**Trigger condition:** `match.uiStatus === "open"` AND `(kickoff_at - now) < 86_400_000ms` (24h).

**Visual placement:** Below the existing date/time line in the match card's bottom row. The bottom row currently contains: `[kickoff date/time] [Enter result button (admin only)]`. The countdown inserts between these two — left-aligned, same row as the date string.

**Exact layout (bottom row):**
```
<div className="flex items-center justify-between">
  <div className="flex flex-col gap-1">
    <span className="text-[6px] text-muted-foreground font-mono">{date} · {time}</span>
    {locksInLabel && <span className="text-[6px] text-pixel-red">{locksInLabel}</span>}
  </div>
  {/* admin enter result button if applicable */}
</div>
```

**Label format:**
- When `h > 0`: `"Locks in {h}h {m}m"` — e.g. "Locks in 3h 42m"
- When `h === 0`: `"Locks in {m}m"` — e.g. "Locks in 47m"
- When `diff <= 0` or `diff > 86_400_000`: no label rendered

**Color:** `text-pixel-red` — creates urgency, distinct from all other text in the card.

**Update interval:** `useEffect` with `setInterval` at 60_000ms (60 seconds). Stores a `tick` counter in state; the `matches` memo depends on `tick` so it recomputes every minute. Clean up with `clearInterval` on unmount.

**No countdown on SAVED rows:** `uiStatus === "saved"` rows show no countdown label. The user already has a prediction; the urgency indicator is irrelevant. (Source: CONTEXT.md specifics)
