<objective>
Provide a fallback mechanism for match results when the Football-Data API is
unavailable or wrong. A global admin signs in with env-var-configured
credentials and manually enters scores. The override triggers immediate
re-scoring and leaderboard recalculation.

**Pivot history**: this plan originally implemented admin via a DB-flag
(`users.is_global_admin`) per DEC-016/017. After team discussion on 2026-05-13,
the env-var branch of D-08 was chosen instead — admin lives in an isolated
subtree of `apps/web` and authenticates without consulting the DB. DEC-018
supersedes DEC-016/017. See `03-CONTEXT.md::D-08-LOCKED` and STATE.md.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → /api/admin/login | Email + password sent in clear over (locally) http. In prod must be https. |
| Client → /api/admin/match-result | Bearer JWT must validate and have `sub: admin`. |
| Admin JWT signing | HS256 with `ADMIN_JWT_SECRET` — secret must be ≥32 random bytes. |
| Admin password storage | bcrypt hash in env file (gitignored locally; secrets manager in prod). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Elevation of Privilege | Admin API | mitigate | `requireAdminToken` verifies an HMAC-signed JWT; payload must have `sub === 'admin'`. Token is only issued after bcrypt.compare against `ADMIN_PASSWORD_HASH`. No DB consulted — no risk of stale/wrong-flag elevation. |
| T-03-04b | Tampering | Admin JWT | mitigate | HS256 signature, server-known secret. JWT tampering rejected by `jwt.verify`. |
| T-03-04c | Information Disclosure | Admin creds in repo | mitigate | `.env` is gitignored; `.env.example` ships placeholder values only. Password is bcrypt-hashed, never plaintext. |
| T-03-04d | Repudiation | Admin actions | accept | Single shared admin identity for v1 — no per-user audit. Acceptable for a single internal operator. Revisit when admin team grows. |
</threat_model>
